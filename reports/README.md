## Reports

### Purpose
This directory contains generated reports from various tools and scripts. Each subdirectory represents a specific report type.

### Structure
- `daily/` - Daily operational summaries
- `audits/` - System health checks
- `tasks/` - Task completion records
- `logs/` - Debug/output logs

### How to Use
1. Use `memory_search` to find relevant report patterns
2. Run `sessions_history` to review session-based reports
3. Check `projects/` for automated report generation scripts

For report templates, see `scripts/README.md` and `notes/README.md`.
